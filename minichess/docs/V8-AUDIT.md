# Gardner MiniChess v8 audit

## Patch scope

v8 is a web/engine patch over v7.1. It does not contain generated tablebase data. Copy the patch over the v7.1 project and keep the existing `tools/gardner_tablebase/tables/` directory in place.

## Browser tablebase integration

The new `js/engine/tablebase.js` probes the Gardner-specific database generated by the v7/v7.1 Python tools. It supports:

- exhaustive GardnerTB WDL+DTM blocks (`manifest.json`);
- verified sparse practical blocks (`practical-manifest.json`);
- material and board-symmetry canonicalization compatible with the Python writer;
- gzip decompression in the analysis/play Web Workers;
- per-material metadata loading and a bounded LRU block cache;
- exact child probing to select shortest wins, longest resistance, and drawing continuations.

The expected directory remains:

```text
tools/gardner_tablebase/tables/
├── manifest.json
├── practical-manifest.json          # optional
├── KQvK/...
├── K.../...
└── practical/...                    # optional
```

A missing table, missing practical record, unsupported browser decompressor, or unproved sparse continuation returns `unknown` and normal Alpha-Beta search continues. No guessed tablebase move is manufactured.

The generated format is GardnerTB/GardnerPracticalTB, not orthodox Syzygy `.rtbw/.rtbz` data.

## Draw and search changes

- Repetition checks remain ahead of the transposition-table lookup.
- When the root side is worse, moves that re-enter a historical position are searched first. Their score is unchanged; this only proves perpetual/threefold resources sooner.
- Failed bounded mate/fortress side probes are memoized for the same position and budget so continuous iterative deepening does not repeatedly pay the same cost.
- Locked two-wing structures with equal directly blocked pawns and a lone opposite-colour bishop are strongly draw-scaled until a pawn move or capture actually breaks the wall. This removes the reported temporary `Bf6 -4.3` horizon spike without declaring a formal tablebase draw.

Regression FEN:

```text
5/3b1/p1k1p/P3P/1K3 b - - 0 1
```

## Mobile layout

At phone width:

- board size is approximately 64% of viewport width and is forced to a 1:1 aspect ratio;
- a narrow White/Black evaluation rail is placed to the left of the board;
- the full page is fixed to `100dvh` with no document scrolling;
- Analysis, Book, and Edit remain visible;
- only the best two engine lines are shown;
- Moves starts collapsed and expands inside the fixed layout;
- controls and play settings use compact, non-clipping grids.

## Validation performed

- all existing core/search/worker/cache/play-mode suites;
- locked-position draw-stability regression;
- exact GardnerTB KQvK probe and full mate-PV replay;
- practical sparse-block index/value decoding against the Python `PracticalTablebase` reader;
- JavaScript syntax, HTML structure, and CSS parse validation.
