# RIES v6.4 changelog

## Algebraic-number recognition

- Added an exact BigInt LLL fallback for algebraic polynomial relations.
- The previous floating Gram-Schmidt lattice reduction could miss true low-height relations at high decimal scales. v6.4 runs exact rational Gram-Schmidt on the key 28-digit lattice slice for degrees up to 8, then verifies candidates against the full available input precision.
- Algebraic rows now show a high-precision polynomial residual `|P(input)|`, not only the double-precision root distance.
- Confirmed examples:
  - `2.4640881167158761343811314520921216123962895091346614` returns `4x^6 + 3x^2 + 16x − 953 = 0` with `|P(input)| ≈ 2.208*10^-49`.
  - `-3.024683580876626972641750062783166137048734369693167842686296157876000573298591` returns `3x^7 + 3x^5 + 19x^2 + 7534 = 0` with `|P(input)| ≈ 1.297*10^-74`.

## Integer shortform database

- Added `assets/shortform100k.js`, an offline precomputed table for `1 <= n <= 100000`.
- The table was generated by exhaustive dynamic enumeration over one- and two-digit constants and total digit budgets 2 through 5, including arithmetic combinations, powers, factorials, binomial coefficients, exact divisions, and nearby `10^5-k` forms.
- The current table covers 95,954 values up to `10^5` with a digit count no worse than the direct decimal expansion.

## UI and result display

- Continued fraction output now has copy buttons for both preview and full expansion.
- Log-combination matches default to two rows.
- Replaced the decorative progress art with a simple rotating tesseract projection.
- Progress updates are monotone within a search run and keep a stable color palette across phases.
