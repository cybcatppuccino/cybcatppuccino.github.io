# Rigid Double Pendulum Web Simulator

A minimal, offline HTML/CSS/JavaScript simulator for a rigid double pendulum with a horizontally moving support point.

## Features

- White full-page background with the whole pendulum system centered on screen.
- Slightly longer horizontal support rail, drawn as a gradient line with no tick marks.
- The rail, first rod and second rod use different blue color tones.
- The viewport transform is based on the full possible sweep of both rods, so the pendulum remains inside the screen for every angle and support position.
- Fully rigid two-link double pendulum using angular generalized coordinates, so rod lengths are exactly constrained.
- The support point moves only on a fixed horizontal segment.
- Internal controller can only command support acceleration, bounded by the user-selected maximum acceleration.
- When the support reaches the segment edge, it is collision-stopped and its velocity becomes zero.
- Four target buttons:
  - State 0 · Down: both rods vertically downward.
  - State 1 · V: first rod downward, second rod upward.
  - State 2 · Lambda: first rod upward, second rod downward.
  - State 3 · Upright: both rods vertically upward.
- Pause/Resume button.
- Friction / damping slider, initially `0.00`.
- `[0,1]^2` angle phase portrait in the upper-left corner:
  - Downward angle maps to coordinate `1/4`.
  - Upright angle maps to coordinate `3/4`.
  - States 0, 1, 2 and 3 are marked at `(1/4,1/4)`, `(1/4,3/4)`, `(3/4,1/4)`, `(3/4,3/4)`.
  - Trail segments are broken at all periodic seams, so no long line is drawn when the point wraps from top to bottom, bottom to top, left to right, or right to left.
  - The trajectory fades out after a short delay.
  - Drag the current phase point to change the two angular positions. During drag, simulation pauses automatically; after release it restores the previous run/pause state. Angular velocities are intentionally preserved.
- Gravity and smooth zero-mean wind perturbation sliders.
- A top-right wind indicator shows the current wind direction and instantaneous wind acceleration.
- The old five-line readout has been removed from the control panel.
- No external libraries or network access required.

## Algorithm updates in this version

### v5 strengthening

- State 0 now uses fast energy-dissipation control when far from the downward equilibrium instead of relying only on gentle local PD stabilization. This makes switching from state 1, 2 or 3 back to state 0 much faster.
- The state-0 CEM controller uses a state-aware deterministic seed: V, Lambda, upright and arbitrary dragged poses start from different exploration basins.
- State 3 uses a repeatable fast CEM seed that reaches the upright capture region much sooner from the default hanging state.
- Target switching now passes the current state into the controller, so the controller can choose an appropriate state-0 swing-down strategy.
- The canvas includes a labeled wind-direction arrow and instantaneous wind-acceleration readout.

### v4 changes

- State 0 uses a dedicated downward-equilibrium stabilizer with a wider quiet zone, so the exact hanging state stays exactly still and the controller avoids injecting energy around the stable point.
- A critical LQR gain bug was fixed. The previous code accidentally kept only the first element of the 1×6 LQR row vector; the new code feeds back both angles, both angular velocities, support position and support velocity.
- The LQR linearization now uses the same RK4 transition map as the simulator instead of a first-order Euler approximation. This makes the local balancer much closer to the actual browser simulation.
- The unstable states 1, 2 and 3 use retuned local LQR gains with stronger but bounded cart-position and cart-velocity penalties.
- The simulation loop uses a fixed small RK4 time step.
- The model-predictive swing-up predictor uses RK4 rollouts and a longer deterministic CEM search.
- The CEM/MPC candidate set includes energy-pumping, target-alignment, centering, bang-bang and sinusoidal sequences. Random exploration is generated by a deterministic PRNG, so behavior is repeatable across browser runs.
- The controller contains three rail-centering layers:
  1. a center-return acceleration heuristic,
  2. a stronger center and edge barrier term in the rollout cost,
  3. a final track-safety limiter that blocks outward acceleration near the rail ends.
- Local balancing uses much lighter command smoothing, because the system is acceleration-controlled and aggressive states 1/2/3 need fast corrective acceleration.
- Optional viscous joint damping is applied by the friction slider.
- Numerical invalid-state checks reset safely instead of letting NaN/Infinity values propagate.

## Run

Open `index.html` in a modern browser.

## File layout

```text
index.html
styles.css

  physics.js      rigid double-pendulum equations, angle mapping, damping and RK4 integration

  renderer.js     main canvas drawing
  phase.js        [0,1]^2 angle phase portrait and drag interaction
  ui.js           controls
```

## Physics notes

Angles are absolute angles measured from the downward vertical. The moving support acceleration enters the Lagrange equations as a horizontal inertial term. The wind perturbation is modeled as a smooth zero-mean horizontal body acceleration applied to both pendulum masses. The friction slider adds viscous damping at the pendulum joints; when it is zero, the previous undamped behavior is preserved.
