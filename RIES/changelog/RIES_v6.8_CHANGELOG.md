# RIES v6.8 changelog

- Replaces the CSS-only projected cube with a canvas-rendered 4D hypercube whose rotation is generated by slowly changing SO(4) plane directions at approximately constant angular speed.
- Removes vertex dots from the hypercube projection and adds slow chromatic edge shifts while preserving a fixed square projection area.
- Cleans copy buttons so copied text is math-only where possible. Factorization copy omits timing metadata, and non-math status labels no longer expose candidate-copy buttons.
- Hides the error column for pure integer-result tables, including factorization, precomputed/database shortforms, and local composite-status rows.
- Adds an explicit settings checkbox for external/worker QS or ECM handoff. It is off by default.
- When the external factoring option is off, unresolved large composite cofactors are only reported as composite after local bounded factoring rather than triggering external handoff.
- Hardens the structured integer database stage by adding inner-loop deadline checks and lowering the synchronous database slice, preventing stalls at “Checking precomputed and structured integer database…”.
- Improves mobile layout with narrower controls, safer table overflow, compact status blocks, and fixed-size responsive hypercube rendering.
